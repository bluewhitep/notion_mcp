# 本地 Notion MCP 服务

## 项目简介

本仓库实现了一个本地运行的 Notion Model&nbsp;Context&nbsp;Protocol (MCP) 服务。项目使用 [Notion Python SDK](https://github.com/ramnes/notion-sdk-py) 对接官方 Notion API，以简化 AI 应用访问 Notion 数据库、页面和区块的流程。与官方远程 MCP 不同，本服务完全在本地运行，用户只需提供集成访问令牌（API token）和自身的 Notion 用户标识即可。

与官方 Notion MCP 服务类似，本项目提供了一套通过 HTTP 和 CLI 调用的工具，以便与各种 AI 代理或其它自动化应用集成。我们在设计时参考了官方 MCP 的接口规范与使用方式，并通过文档、模块化代码和充分的测试保证可维护性和扩展性。

## 目录结构概览

项目采用层次化的目录结构，以便于开发和测试。部分关键目录如下：

| 目录               | 说明                                             |
|--------------------|--------------------------------------------------|
| `nilo/`      | 核心源代码，包含配置管理、路由和 CLI 实现等模块 |
| `nilo/routes/` | 不同功能的 API 路由（数据库、页面、区块）      |
| `nilo/models/` | 数据模型定义，便于请求校验和文档生成           |
| `tests/`           | 针对各模块的单元测试和场景测试                  |
| `Docs/`            | 文档目录，包含需求、设计、技术选型和使用指南等 |

## 如何使用

详细的安装与使用说明请参阅 `Docs/EN/User/Guide.md`。简而言之，用户需要先通过 CLI 初始化配置（设置 Notion 集成的 token 和个人的 UUID），然后启动服务即可通过 HTTP 调用接口操作 Notion 数据。关于开发流程和 API 参考，请参阅 `Docs/EN/Developer/API.md`。

## 开发与贡献

项目使用 [FastAPI](https://fastapi.tiangolo.com/) 构建 Web 服务，使用 [Typer](https://typer.tiangolo.com/) 提供命令行界面，使用 [pytest](https://docs.pytest.org/) 编写单元测试。设计文档、需求分析和技术选型文档均存放在 `Docs/` 下，欢迎按照文档内容推进功能开发并提交 PR。

在开发功能时，需遵循以下原则：

1. **先写测试，后写实现。** 所有功能必须先编写对应的测试代码，再编写实现代码。测试未通过时只能修改实现或增加新的测试覆盖，不能修改已有测试逻辑。
2. **文档同步更新。** 新增或修改功能后，需要更新开发者文档和用户文档，记录接口变化和使用方式。
3. **层次化结构。** 代码按照功能拆分至不同模块，测试、文档也按照读者对象（开发者/使用者）拆分，不得混用。
