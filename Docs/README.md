# N.I.L.O. Documentation

N.I.L.O. documentation is organized by language. `Docs/ZH` is the primary source for documentation content decisions; `Docs/EN` and `Docs/JP` keep the same structure for localized reading.

## Languages

- [中文](ZH/README.md)
- [English](EN/README.md)
- [日本語](JP/README.md)

## Structure

Each language tree keeps the same high-level audience split:

- `User` - installation, configuration, CLI usage, MCP client setup, skill templates, and troubleshooting.
- `Developer` - architecture, API notes, tests, and MCP tool contracts.

When updating source documentation content, update `Docs/ZH` first, then synchronize the corresponding `Docs/EN` and `Docs/JP` entries as needed.
